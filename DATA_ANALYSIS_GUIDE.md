## 🔍 분석 가능한 주요 지표

### 1️⃣ **재고 현황 분석**

#### 가용재고 분석
```
분석 항목: availableQuantity (가용 재고)
공식: 가용재고 = 전체재고 - 예약 - 불량 - 반품처리 - 검사

예시:
├─ PROD-00001 (에코 티셔츠)
│  ├─ 전체: 2,500개
│  ├─ 예약: 300개
│  ├─ 불량: 50개
│  ├─ 반품: 150개
│  ├─ 검사: 100개
│  └─ 🟢 가용: 1,900개 (76%)
│
└─ PROD-00002 (데님 팬츠)
   ├─ 전체: 5개
   ├─ 예약: 0개
   ├─ 불량: 2개
   ├─ 반품: 1개
   ├─ 검사: 1개
   └─ 🟢 가용: 1개 (20%) ⚠️

분석 결과:
- 가용재고율 > 70%: 충분한 재고 ✅
- 가용재고율 30-70%: 주의 필요 ⚠️
- 가용재고율 < 30%: 긴급 조치 필요 🚨
```

#### 재고 상태별 분류
```sql
SELECT category, stockStatus, COUNT(*) as count, SUM(quantity) as total_qty
FROM inventory
GROUP BY category, stockStatus
ORDER BY category, stockStatus

결과 예시:
의류     | in_stock      | 8   | 15,200개
의류     | low_stock     | 2   | 55개
신발     | in_stock      | 3   | 5,820개
신발     | low_stock     | 1   | 120개
가방     | in_stock      | 3   | 4,980개
가방     | low_stock     | 1   | 450개
액세서리 | in_stock      | 5   | 9,080개
액세서리 | low_stock     | 0   | 0개
```

## 📈 Excel/데이터시트 분석 방법

### Google Sheets에서 분석

1. **CSV 임포트**
   ```
   1. Google Sheets 열기 (sheets.google.com)
   2. 파일 > 새로 만들기 > 스프레드시트
   3. 파일 > 가져오기 > 업로드 탭
   4. mock-inventory-data.csv 선택
   5. 임포트 옵션: UTF-8, 쉼표 구분 선택
   ```

2. **필터 적용**
   ```
   1. 헤더 행 선택 (첫 번째 행)
   2. 데이터 > 필터 > 필터 만들기
   3. 각 컬럼 헤더의 필터 아이콘 클릭
   4. 원하는 조건 설정
   ```

3. **피벗 테이블 생성**
   ```
   1. 삽입 > 피벗 테이블
   2. 행: category, 값: SUM(quantity)
   3. 카테고리별 재고 분포 확인
   ```

4. **차트 생성**
   ```
   1. 삽입 > 차트
   2. 차트 유형: 원형/막대/꺾은선 등 선택
   3. 데이터 시리즈 설정
   ```

## 🔧 SQL 쿼리 예시

### 1. 가용재고 부족 상품
```sql
SELECT productCode, productName, quantity, availableQuantity, 
       ROUND(availabilityRate, 2) as availRate
FROM inventory
WHERE availabilityRate < 50
ORDER BY availabilityRate ASC;

결과:
├─ PROD-00008: 가용재고 20% (긴급 위험)
├─ PROD-00002: 가용재고 20%
├─ PROD-00004: 가용재고 40%
└─ ...
```

### 2. 카테고리별 재고액 TOP 3
```sql
SELECT category, COUNT(*) as productCount, 
       SUM(quantity) as totalQty, 
       SUM(totalStockValue) as totalValue
FROM inventory
GROUP BY category
ORDER BY totalValue DESC
LIMIT 3;

결과:
├─ 의류: 8개 상품, 15,200개, 389,000,000원
├─ 신발: 4개 상품, 5,820개, 185,140,000원
└─ 가방: 4개 상품, 4,980개, 109,620,000원
```

### 3. 미주문 상품 정리
```sql
SELECT productCode, productName, daysWithoutOrder, 
       quantity, totalStockValue,
       CASE 
           WHEN daysWithoutOrder > 180 THEN '긴급 정리'
           WHEN daysWithoutOrder > 100 THEN '할인 판매'
           WHEN daysWithoutOrder > 50 THEN '마케팅 강화'
           ELSE '정상'
       END as action
FROM inventory
WHERE daysWithoutOrder > 50
ORDER BY daysWithoutOrder DESC;
```

### 4. 위치별 재고 밀도
```sql
SELECT building, SUM(quantity) as total_qty, COUNT(*) as item_count,
       ROUND(AVG(quantity), 0) as avg_qty_per_item
FROM inventory
GROUP BY building
ORDER BY total_qty DESC;

결과:
├─ A동: 10,200개 (8개 상품 평균 1,275개/상품)
├─ B동: 9,400개 (7개 상품 평균 1,343개/상품)
├─ C동: 3,850개 (4개 상품 평균 963개/상품)
└─ D동: 2,850개 (1개 상품 평균 2,850개/상품)
```
